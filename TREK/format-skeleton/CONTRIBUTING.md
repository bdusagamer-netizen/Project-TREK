# Contributing to Trek

Thank you for your interest in contributing to Trek! We welcome contributions from everyone.

## Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally
3. **Create a feature branch** for your changes
4. **Make your changes** and commit with clear messages
5. **Push to your fork** and submit a pull request

## Development Setup

```bash
git clone https://github.com/trek-lang/trek.git
cd trek
npm install
npm run build
npm run test
```

## Code Style

- Use consistent formatting (2-space indentation)
- Write meaningful commit messages
- Add comments for complex logic
- Follow existing patterns in the codebase

## Reporting Bugs

1. Check if the issue already exists
2. Provide a clear description
3. Include a minimal reproduction example
4. Specify your OS and version information

## Feature Requests

We love feature ideas! Please:
1. Describe the feature clearly
2. Explain the use case and benefit
3. Provide examples if possible
4. Discuss potential implementation approach

## Documentation

Help improve Trek documentation:
- Fix typos and improve clarity
- Add examples and tutorials
- Translate documentation
- Improve API documentation

## Testing

All contributions should include tests:
```bash
npm test
```

## Pull Request Process

1. Update documentation if needed
2. Add tests for new functionality
3. Ensure all tests pass
4. Submit PR with clear description
5. Address review feedback

## Code of Conduct

### Our Pledge

We are committed to providing a welcoming and inspiring community for all.

### Our Standards

Examples of behavior that contributes to creating a positive environment include:
- Using welcoming and inclusive language
- Being respectful of differing opinions
- Accepting constructive criticism
- Focusing on what is best for the community
- Showing empathy towards other community members

### Enforcement

Instances of unacceptable behavior may be reported by contacting the project team. All reports will be reviewed and investigated.
